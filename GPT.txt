ChatGPT:
Нет, PostgreSQL в config.yml Cloudflare Tunnel добавлять не надо и нельзя.

Tunnel — это только публикация HTTP/HTTPS сервисов наружу:

Интернет
   |
Cloudflare Tunnel
   |
frontend :3000
backend  :8000

Postgres остаётся внутри Docker-сети:

backend container
       |
       |
postgres container

Он вообще не должен быть доступен из интернета.

Что проверить

В проекте сделай поиск:

192.168.0.113

и:

8000

Через VS Code:

Ctrl + Shift + F

Ищи:

192.168.0.113:8000

Скорее всего найдёшь что-то типа:

NEXT_PUBLIC_API_URL=http://192.168.0.113:8000

или:

const API_URL = "http://192.168.0.113:8000"
Почему после изменения env не сработало

С Next.js есть важный момент.

Если переменная имеет:

NEXT_PUBLIC_API_URL

то она вшивается в JS при сборке.

То есть ты мог:

поменять .env
перезапустить контейнер
но старый frontend build остался

Нужно именно:

docker compose down
docker compose build --no-cache frontend
docker compose up -d

или если сервис называется иначе:

docker compose build --no-cache
docker compose up -d

https://api.familytreebp.ru/v1/user/login
