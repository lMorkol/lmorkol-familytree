# AI RULES (CRITICAL)

## 1. Работа с кодом

- всегда проверяй существующий код перед созданием нового
- не создавай дублирующую логику
- используй текущую архитектуру

---

## 2. Изменения

- сначала объясни что хочешь изменить
- потом предложи план
- только затем код

---

## 3. Ограничения

- не добавляй новые слои без необходимости
- не усложняй архитектуру
- не предлагай микросервисы без явного запроса

---

## 4. Поведение

- если не уверен — задай вопрос
- не выдумывай детали проекта
- не предполагаешь скрытые модули

---

## 5. Код

- изменения должны быть минимальными
- предпочтение — правке существующего кода, а не переписыванию

## IMPORTANT

- не изменяй существующие модели данных без явного запроса
- всегда проверяй зависимости перед изменением кода
- не создавай новые сущности если можно расширить существующие
