# NON-FUNCTIONAL REQUIREMENTS

## Performance
- API должен отвечать < 300ms (p95)
- тяжелые операции не блокируют основной поток

---

## Reliability
- система не должна падать из-за ошибок внешних сервисов
- ошибки логируются, но не ломают процесс

---

## Security
- пароли хранятся только в хэшированном виде
- JWT используется для авторизации
- доступ к API контролируется ролями

---

## Maintainability
- код должен быть расширяемым
- запрещено дублирование бизнес-логики

---

## Observability
- логируются:
  - ошибки
  - ключевые бизнес-операции
