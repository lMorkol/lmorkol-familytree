# PROJECT CONTEXT

## 1. Назначение системы
Кратко: что делает проект и для кого

## 2. Бизнес-логика
Данные проект предназначается для того, чтобы люди могли сохранить историю своего семейного древа. Чтобы потомки могли узнать о том, кем были их предки, как они жили, чем увлекались и какие события происходили в их семье.

## 3. Архитектура (текущая)
- backend: python, fastapi
- frontend: React / Next.js
- database: PostgreSQL
- external services: Docker

## 4. Основные сущности
User:
- Пользователь(user), который проходит регистрацию и авторизацию. Может создавать деревья и людей в рамках них (если достаточно прав). Может иметь отношение к нескольим деревьям с разным уровнем прав.
- Дерево(tree) - контейнер, который хранит в себе информацию о людях(human)
- Человек(human) - контейнер информации о человеке, может хранить фото, текст, важные события, браки, место работы, пол, дату рождения, место жительства, гражданство.
- Событие(event) - описание какого-либо события в жизни человека. Имеет отношение строго к одному человеку.

## 5. Основные сценарии
1. Я как пользователь, создаю дерево семьи, чтобы сохранить историю рода.
2. Я как пользователь, создаю человека, чтобы сохранить историю, конкретного лица.
3. Я как пользователь, заполняю карточку человека, чтобы сохранить основную информацию о нём.
4. Я как пользователь, создаю событие, чтобы зафиксировать важный момент в жизни человека.
5. Я как пользоватлеь, хочу иметь возможность добавлять в дерево других пользователей, чтобы мы могли совместно работать над реставрацией и сохранением истории.


## 6. Ограничения системы
- Прикрепить человека к двум деревьям

## 7. Что считается “правильной реализацией”
- минимальные изменения
- переиспользование существующего кода

## 8. DATA MODEL RULES

- одна сущность = одна таблица в БД
- связи явно определяются (1-N, N-N)
- нельзя выдумывать поля без запроса
- все изменения схемы должны быть согласованы с существующими API
