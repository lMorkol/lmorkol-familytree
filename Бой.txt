Ты senior backend engineer и DevOps.

Проанализируй весь проект перед внесением изменений.

Цель — подготовить проект к публикации в интернете, не меняя бизнес-логику.

Не переписывай существующий код без необходимости.

Нужно выполнить следующие задачи.

1. Проверить безопасность проекта.
- убедиться, что секреты не находятся в git
- перенести все секреты в .env
- добавить .env.example
- проверить JWT
- проверить CORS
- проверить настройки PostgreSQL

2. Проверить Docker.
- docker-compose должен запускать весь проект одной командой
- добавить healthcheck если отсутствуют
- использовать restart unless-stopped
- убедиться, что backend не зависит от localhost

3. Подготовить production-конфигурацию.
- проверить переменные окружения
- убрать debug режим
- добавить корректное логирование
- проверить обработку ошибок

4. Проверить frontend.
- убедиться, что API URL берется из переменной окружения
- убрать localhost из кода
- проверить production build

5. Подготовить проект к деплою на VPS.

Если отсутствуют, создай:

docs/deployment.md
.env.example
.gitignore (при необходимости)

В deployment.md подробно опиши:

- как установить Docker
- как запустить проект
- как обновлять проект
- как обновлять контейнеры
- какие команды использовать

6. Проверить структуру проекта.

Не создавать новые папки без необходимости.

Не менять архитектуру.

Не менять API.

Не менять схему базы данных.

7. После анализа выведи отчет в формате:

## Что найдено

...

## Что нужно исправить

...

## Что будет изменено

...

После этого дождись моего подтверждения.

Только после подтверждения приступай к изменениям.
