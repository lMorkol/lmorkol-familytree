Нужно подготовить проект к работе через публичный домен Cloudflare Tunnel.

Проблема:
Frontend сейчас отправляет API-запросы на:
https://familytreebp.ru:8000/v1/user/login

Это неправильно. Frontend должен работать через основной домен:
https://familytreebp.ru

Backend должен быть вынесен на отдельный поддомен:
https://api.familytreebp.ru

Задача:
1. Найди в проекте все места, где указан URL backend API:
   - localhost
   - 127.0.0.1
   - familytreebp.ru:8000
   - любые другие захардкоженные адреса API

2. Определи текущий механизм настройки API URL:
   - env файлы
   - конфигурация Next.js/React
   - переменные окружения Docker
   - другие настройки

3. Предложи минимальный набор изменений для перехода на:
   API_BASE_URL=https://api.familytreebp.ru

4. Не меняй архитектуру проекта.
5. Не переписывай компоненты без необходимости.
6. Не меняй backend.
7. Сначала покажи:
   - какие файлы будут изменены
   - что именно будет изменено
   - почему

После моего подтверждения внеси изменения.

Дополнительно проверь:
- не используется ли API URL напрямую внутри кода;
- не требуется ли изменение docker-compose;
- не требуется ли изменение .env.production или аналогичного файла.

Цель:
После изменений frontend должен обращаться только к:
https://api.familytreebp.ru
а локальная разработка должна сохранить возможность работы через localhost.
