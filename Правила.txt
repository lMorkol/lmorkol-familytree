Создай docs/05-ai-rules.md.

Документ предназначен исключительно для AI-ассистентов.

Включи следующие правила.

Перед изменением кода:

- сначала изучить существующий код;
- не создавать дубли;
- переиспользовать существующие сервисы;
- соблюдать текущую архитектуру;
- не менять публичное API без необходимости;
- не менять структуру базы данных без согласования;
- сначала предложить план изменений;
- затем дождаться подтверждения;
- после этого писать код.

Во время написания кода:

- писать минимальные изменения;
- придерживаться существующего стиля;
- использовать существующие зависимости;
- не создавать новые слои без необходимости;
- не усложнять архитектуру;
- писать читаемый код.

При сомнениях:

- задавать вопрос;
- не придумывать отсутствующие сущности;
- не делать предположений.

Документ должен быть коротким, строгим и понятным для AI.
